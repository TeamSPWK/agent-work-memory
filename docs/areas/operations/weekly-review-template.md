# Weekly Review Template

매주 30분 회고. **사본을 만들지 말고 본 파일에 누적** (역시간순 — 최신이 위).

## 사용법

1. 매주 1회 (예: 금요일 오후 또는 월요일 오전)
2. 본 파일의 "## 누적 회고" 섹션 맨 위에 새 회고 블록 추가
3. NOVA-STATE.md *Last Activity*와 교차 정합 (큰 결정은 양쪽에 기록)
4. 깨진 싱크 발견 시 *"## 8 적용 대상"* (`.claude/rules/operations-sync.md`)대로 처리

## 회고 블록 양식

```
## YYYY-MM-DD Week N

### 결제·매출
- MRR: ___ (전주 대비 ±___)
- 신규 결제: ___ / 해지: ___

### 고객 / 인터뷰
- 새 인터뷰: ___
- 디자인 파트너 상태 변화: ___

### 비용·운영
- Cloud 청구서 변동: ___
- 사고/장애: ___

### 제품
- 출시 기능: ___
- 다음 주 우선순위: ___

### 깨진 싱크
- ___ (정합성 스크립트 결과 또는 수기 발견)

### 핵심 결정 1문장
- ___
```

## 회피

- 이슈 없는 항목은 "—"로 표기. 빈 칸 두지 말 것 (다음 주 누가 봤을 때 "안 적은 건지 없는 건지" 모호).
- 회고 블록 누적이 너무 길어지면 분기별로 `weekly-review-archive-YYYYQ.md`로 잘라낸다.
- 매일 회고 X. 매주 1회 한도.
