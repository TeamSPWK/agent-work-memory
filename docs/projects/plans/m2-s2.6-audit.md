### 탭 라우팅 패턴 (결정 #1)

**선택**: `useSearchParams()` + 단일 `<Audit />` 컴포넌트. 4 child route 미생성.

```tsx
// screens/Audit.tsx
import { useSearchParams, Navigate } from 'react-router-dom'
import { AUDIT_TABS } from '@/lib/seed/navigation'

export function Audit() {
  const [params, setParams] = useSearchParams()
  const tab = params.get('tab') ?? 'trail'
  if (!AUDIT_TABS.find(t => t.id === tab)) return <Navigate to="/audit" replace />
  return (
    <>
      <AuditHeader />              {/* page-h + 공통 actions */}
      <TabBar active={tab} onChange={(id) => setParams({ tab: id })} />
      {tab === 'trail' && <AuditTrail />}
      {tab === 'principles' && <Principles />}
      {tab === 'integrity' && <Integrity />}
      {tab === 'pdf' && <PdfPreview />}
    </>
  )
}
```

근거:
- `projectStatus.ts`가 이미 `?tab=` 패턴으로 화면 매트릭스 등록 → 도큐먼트 결정 반영.
- 4 sub-route를 만들면 `router.tsx`가 비대. SessionDetail은 형제 화면(ExplainBack/Share/SelfRecall)이 *독립 페이지*라 분리한 것과 다른 케이스.
- 새로고침·딥링크는 쿼리로 유지됨 (router state 아님).

**안 채택**:
- Nested route (`/audit/trail`, `/audit/principles`) — 4 child route 추가가 단일 KPI 헤더 공유 패턴과 어긋남.
- 클라이언트 state only — 새로고침 시 trail로 리셋되어 H2 시연(딥링크 공유)이 깨짐.

### Sub-screen 분리 (결정 #2)

`h2-audit.jsx`의 5 함수 중 4개를 1:1 분리. 공통 헤더(`page-h`)는 `Audit.tsx`의 `<AuditHeader />`에서 actions만 탭에 따라 가변(예: `trail`만 "체인 검증·PDF export" 버튼, `principles`는 "감사 자료 PDF").

각 sub-screen은 디자인 jsx에서 *내용*만 가져오고 styling은 `global.css` 토큰 재사용. 새 클래스 추가 금지.

### Billing 버튼 처리 (결정 #3)

`h2-audit.jsx:216, 411~559`의 `gotoSession("billing")` 버튼은 S2.6 범위 외. 대신:
- `Principles.tsx` Starter cta의 "업그레이드 →" → `<Link to="/settings?tab=export">업그레이드 →</Link>` (placeholder로 진입).
- `Audit.tsx` 헤더의 actions에는 Billing 버튼 미배치.

S2.10에서 `/settings?tab=export`가 BillingScreen 내용을 흡수할 때 연결됨.
