### 핵심 결정

1. **시드 분리**: WS_MEMBERS는 Reviewer 화면 5명만 발췌. 풀 시드(role matrix·KPI 등)는 S2.9 Workspace에서 완성. *cross-sprint 데이터 정합성*은 S2.9 시점에 검증.

2. **OnboardingProgress 두 곳 표시 정합**:
   - 디자인은 *상단 layout 헤더 NavLink 5* + *각 화면 상단 inline bar* 둘 다.
   - 본 구현도 둘 다 유지 — layout NavLink는 *라우팅 진입점*, inline bar는 *현재 step 시각화*. 중복 X, 정보 다름.

3. **Connect OAuth 모달**: dialog overlay + Escape/backdrop click close. `useState<ToolKey|null>`. 4 tool state local — `connected/idle/error` 3가지. 모달 허용 시 setTools 적용.

4. **Reviewer 동적 §1**: `picked.length === 0` → "Reviewer 미지정 — pending" / 1+명 → "활성". 디자인 line 477~501 정합.

5. **Done Today 점프**: 디자인은 window.dispatchEvent. production은 `<Link to="/today" className="btn primary lg">Today 열기 →</Link>`. *AppShell 외부(OnboardingLayout)에서 진입*하므로 풀 라우터 navigation.

6. **mock 한계 배지**: Done 화면의 "4분 38초"는 hardcoded 시연용 — *디자인 sub: '시연용 mock'* 그대로 옮기되 추가로 PROTOTYPE_MARKS #13 추가 (실측은 S5+ 디자인 파트너 실 진입 시).

7. **5분 가설 측정**: 본 sprint는 *시각만* 채움. 실 측정은 S9 (`H4 온보딩 5분 측정`) — 별도 sprint.
