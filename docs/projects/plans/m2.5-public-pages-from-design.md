# M2.5 — 외부 페이지 시안 → 코드 이식

> v0.2 디자인 시안 14화면(`docs/projects/plans/p0-design-v0.2/`)을 production 코드로 옮기는 단계. m2(inside-app)와 같은 stack 위에 *public route* 트리를 분리해 얹는다.
> CPS 구조. exit criteria 명시.

## Context

- **2026-05-11 v0.2 lock**: claude.ai/design 누적 세션 3차 라운드에서 외부 14화면 시안 받음(랜딩·가격·가입 3종·법무 4종·회사·에러 3종·상태). chat-3.md에 10개 결정 기록.
- **m2 결정 stack 그대로**: Vite+React+TS · Supabase Tokyo · Supabase Auth · Vercel · 토스페이먼츠 · PopBill. 외부 페이지도 동일 stack에서 *별도 라우트 트리*로 작동.
- **PRD v2.1 §12** 외부 페이지 영역 명문화 완료. 법무 4종 문구는 별도 트래커(`docs/areas/regulatory/legal-pages.md`)에서 추적.

## Problem

m2(inside-app)와 m2.5(public)는 같은 stack을 공유하지만 다음 차원에서 충돌·분리가 발생한다:

| 차원 | 충돌 / 필요 |
|---|---|
| **라우터 트리** | `/app/*` (authenticated) vs `/`·`/pricing`·`/login` 등(public) 분리 필요 |
| **Layout** | `Shell`(sidebar + persona + workspace) vs `PublicShell`(top-bar + footer) |
| **인증 경계** | public route는 미인증 OK, `/app/*`는 Supabase Auth 필수 |
| **SEO** | public은 SEO 의미 있음 (랜딩·가격·회사). inside-app은 noindex |
| **OG / 메타 태그** | public은 카카오톡·X·LinkedIn 공유 시 thumbnail. inside-app은 비공개 |
| **sitemap.xml** | public 페이지 13개만 (404·500·유지보수 제외) |
| **세션 핸드오프** | 가입(public) → H4 화면1(authenticated) — Supabase Auth 직후 라우터가 H4로 점프 |
| **법무 문구** | 시안은 placeholder, 실제 문구는 별도 트래커에서 lock 후 합류 |

## Solution

### 라우터 분리

```
src/routes/
  public/                ← 미인증 OK
    index.tsx           — 랜딩 (path "/")
    pricing.tsx         — 가격 ("/pricing")
    signup.tsx          — 가입 ("/signup")
    login.tsx           — 로그인 ("/login")
    reset.tsx           — 비밀번호 재설정 ("/reset")
    terms.tsx           — 이용약관 ("/legal/terms")
    privacy.tsx         — 개인정보처리방침 ("/legal/privacy")
    refund.tsx          — 환불 정책 ("/legal/refund")
    business.tsx        — 사업자 정보 ("/legal/business")
    company.tsx         — 회사 ("/company")
    status.tsx          — 상태 페이지 ("/status")
    err404.tsx          — 404 (fallback)
    err500.tsx          — 500
    maint.tsx           — 유지보수
  app/                  ← Supabase Auth 필수, m2 범위
    today.tsx, sessions.tsx, ... (v0.1 23화면)

src/layout/
  PublicShell.tsx       — flat top-bar + footer (v0.2 public-shell.jsx 변환)
  AppShell.tsx          — sidebar + topbar + persona toggle (m2 범위)
```

**Route guard**: `<AppShell>` 안의 모든 child가 `<RequireAuth>` 통과해야 렌더. 미인증 시 `/login` 리다이렉트 + `redirectTo` 쿼리 파라미터로 복귀 경로 보존.

`sitemap.xml`: 랜딩·가격·회사·상태 4개만. 법무·가입·에러는 제외.

### 세션 핸드오프 (가입 → H4)

```
1. /signup 에서 이메일·비번 입력 or Google OAuth
2. Supabase Auth signUp() 성공
3. await supabase.auth.signInWithPassword()  // 자동 로그인
4. navigate("/app/onboarding/workspace", { replace: true })  // H4 화면 1
5. (H4 화면 1) 워크스페이스 생성 → memberships 자동 INSERT(role=Admin)
6. 진행 게이지 0/5 → 1/5
```

가입 직전 `localStorage`에 `redirect_after_auth` 저장하지 않음 (가입은 *언제나* H4 화면 1로). 로그인은 redirectTo 쿼리 파라미터 존중.

### 결제 단위 *Active Operator* 정의 4지점 동기화

디자이너 Decision 4가 강조한 *4지점 반복*: 가격 페이지 · 랜딩 §7 미리보기 · 이용약관 §2 · 환불 정책. 코드 이식 시 **single source of truth**(`src/lib/copy/active-operator.ts`)에서 동일 문자열 export 하여 4 페이지가 같이 import. 변경 시 한 곳만 수정.

### 사업자 정보 single source (Decision 9)

```typescript
// src/lib/config/business.ts
export const BIZ = {
  company: "Spacewalk",
  ceo: "jay@spacewalk.tech",
  bizNo: import.meta.env.VITE_BIZ_NO || "",      // 빈 문자열이면 placeholder
  ecommNo: import.meta.env.VITE_ECOMM_NO || "",
  address: import.meta.env.VITE_BIZ_ADDR || "",
  email: "jay@spacewalk.tech",
  channel: "채널톡 · 영업시간 응답 1~2 영업일",
  updated: "2026-05-11",
};
```

`bizNo === ""`이면 footer와 사업자 정보 페이지 양쪽 모두 `[사업자 등록 후 입력]` 회색 placeholder 렌더. 사업자 등록 완료 시 Vercel env에 `VITE_BIZ_NO` 추가하면 두 곳 동시 동기화.

### 법무 4종 문구 합류 흐름

```
docs/areas/regulatory/legal-pages.md 단계 1~5 완료
  ↓
public/legal/*.md (또는 src/content/legal/*.md) 한국어 원문 lock
  ↓
m2.5 cli/scripts/check-legal-sync.ts — markdown 헤더와 사이트맵 매칭 자동 검증
  ↓
변경 시 footer "마지막 개정 2026-XX-XX" 자동 갱신
```

코드 이식은 placeholder 렌더 + markdown content slot 으로 시작. 실제 문구가 들어오면 markdown만 교체.

## 작업 순서 (m2 의존성 기반)

m2.5는 m2 S1~S3 완료 후 진입. *S4 이후*는 병렬 가능.

```
M2.5 / S1. Public route 골격
    src/routes/public/* 스캐폴드, PublicShell layout, 라우터에 등록.
    14 페이지는 비어 있어도 OK — 라우트 dispatch만 검증.
    → Exit: localhost에서 14 페이지 navigate 통과

M2.5 / S2. v0.2 컴포넌트 → TSX 변환
    public-landing/pricing/auth/legal/company/error/status 7개 JSX → 14 TSX 분리.
    스타일은 v0.2 styles.css 그대로 import. 토큰 변경 없음.
    → Exit: criteria #1 충족 (시각 검증) — 단, 정적 콘텐츠만

M2.5 / S3. 라우트 가드 + 세션 핸드오프
    <RequireAuth>, /login redirectTo, 가입 → H4 화면1 점프.
    → Exit: criteria #2, #3 충족

M2.5 / S4. SEO + sitemap + OG
    react-helmet-async 또는 동급. sitemap.xml 생성 스크립트.
    → Exit: criteria #4 충족

M2.5 / S5. PUBLIC_BIZ env-derived + placeholder
    src/lib/config/business.ts, env wiring, placeholder fallback.
    → Exit: criteria #5 충족

M2.5 / S6. Active Operator copy single source
    src/lib/copy/active-operator.ts, 4 페이지 import 교체.
    → Exit: criteria #6 충족

M2.5 / S7. 법무 markdown slot + sync 검증
    src/content/legal/*.md placeholder, terms/privacy/refund/business 페이지에서 import.
    scripts/check-legal-sync.ts.
    → Exit: criteria #7 충족 (placeholder만 — 실제 문구는 별도 트래커)

M2.5 / S8. 라이트/다크 시각 + footer 점검
    PublicFooter 다크 모드 가독성, hover 색.
    → Exit: criteria #8 충족

M2.5 / S9. /nova:check 전체 — 8 criteria PASS 후 m2.5 완료
```

## 위험 + 대응

| 위험 | 가능성 | 영향 | 대응 |
|---|---|---|---|
| 가입 직후 H4 점프 race condition (auth state hydration 미완) | 중 | 중 | onAuthStateChange listener 도착 후 navigate. 빈 화면 spinner |
| 법무 markdown 직접 commit으로 sync 깨짐 | 중 | 중 | `check-legal-sync` CI에 강제. 자동 PR 댓글 |
| SEO 메타가 client-side 렌더로만 — 검색엔진 인덱싱 약함 | 중 | 중 | Vercel Static 빌드 시 페이지별 prerender(`vite-plugin-ssr` 또는 동급 — *m3* 이후 결정. m2.5는 기본 meta tag만) |
| 사업자 정보 미입력 placeholder가 *프로덕션에 노출* | 낮 | **치명** | Vercel env에 `VITE_BIZ_NO` 미설정 시 빌드 fail 정책 추가 — 단, **개발자 모드만**. 디자인 파트너 페이지는 placeholder가 의도된 디자인이므로 강제 fail 아님 |
| 디자인 파트너 50% chip 노출이 *영구*로 보임 (시간 기준 milestone X) | 낮 | 낮 | "선착순 5팀 한정" 카피 + 카운트(5/5 vs 3/5 등) 동적 표시. 5팀 채워지면 chip 자동 숨김 |
