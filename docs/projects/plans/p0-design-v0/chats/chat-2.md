# Chat 2 — v0.1 보강 라운드 디자인 결정 기록

## 라운드 목표
v0(claude.ai/design)에서 16화면으로 H1·H2·H3 핵심 가설을 시각화한 뒤,
v0.1에서는 의도적으로 비어 있던 보조 화면을 채웠다. 추가 범위는 네 묶음:

1. **H4 · 5분 first-value 온보딩** (5화면, 가설 검증 띠 추가)
2. **Workspace** (Members / Invite / Roles & Risk 매트릭스) — 지원 화면
3. **Settings** (Profile / Integrations / Notifications / Audit Export) — 지원 화면
4. **Risk Radar 비-DB 7 카테고리** 시그널 리스트 채움

총 신규 페이지 12개 + Risk Radar 카테고리 7개 = 19개 단위가 v0에 합쳐졌다.

## 시각·인터랙션 결정

- **사이드바 그룹 라벨 구분.** H1·H2·H3·H4는 가설 라벨(`Hn · ...`)을, Workspace·Settings는 도메인 라벨을 그대로 사용. 가설 띠(상단 hypothesis banner)는 H1~H4에만 표시하고, Workspace·Settings에서는 회색 톤의 *지원 화면 — 상시 사용 · 가설 검증 대상 아님* 띠로 교체. 사용자는 시연 흐름과 운영 화면을 한눈에 분리할 수 있다.
- **H4 진입 시 페르소나 자동 스왑.** 가설이 바뀌면 페르소나가 해당 가설의 primary persona로 자동 갱신되도록 v0 라우터 로직을 그대로 재사용. H4 활성 시 Admin으로 변경.
- **H4 진행 게이지.** 화면별 1/5 → 5/5 표기를 우상단 KPI 카드로 고정. 5화면 모두 동일 위치에서 단계 추적 가능.
- **Roles & Risk 매트릭스.** 위험 카테고리 8 × 역할 3 = 24셀. 각 셀은 *승인 / 검토 / view / —* 4 chip 중 하나. 1셀이라도 변경 시 audit log 기록 — 변경 사유 입력 권장 알림을 매트릭스 하단에 고정.
- **Notifications 무음 시간대.** 1인 운영 sustainability를 카피로 명시("무음 시간대 동안은 신규 신호가 누적된 뒤 09:00에 묶음으로 통지"). 24/7 지원·즉시 알림 가정을 디자인 수준에서 차단.
- **Audit Export · SHA-256 변경 불가.** 변조 불가 설계의 핵심임을 카드 안에 *변경 불가* chip + 카피로 못 박았다. 보존 기간은 5년 기본 + 7/10/영구 옵션에 비용 영향 한 줄 첨부.
- **Risk Radar 7 카테고리.** v0 mock 사건과 cross-link 가능한 식별자(s-014, s-021, s-022, INC-26-014 등)를 그대로 재사용. 카테고리별 신호 테이블은 *시각 / 이벤트 + 명령 + 운영 노트 / 세션 / repo / 심각도 / detail →* 6열 구조로 통일했다. DB 카테고리는 v0 인라인 표를 그대로 유지(특별 케이스).

## anonymization·1인 운영 제약 자가 점검

- 가상 회사·인물명 0건. 멤버는 *대표 (CTO 겸직) · 디자인 리드 (6년차) · 운영 매니저 (4년차)* 식 역할+연차만.
- 24/7 지원·enterprise sales·매뉴얼 운영 카피 0건. Notifications 무음 시간대 + 초대 메일 카피("답변은 영업일 기준 1~2일 안에")로 명시.
- 임의 기간 milestone 0건. 단 *5년 법정 권고*는 인공지능기본법 보존 기준 인용이며 임의 기간 아님.
- 원문 transcript 미저장 카피는 초대 메일·H4 화면 2·Members 안내 등 4지점에서 반복.
