# Claude Design — v0.1 보강 프롬프트 (Round 2)

> v0(claude.ai/design) 결과를 base로 받고 누락 화면 + Risk Radar 카테고리를 추가하는 2차 라운드.

## 사용 방법

1. claude.ai/design 새 세션 시작
2. **v0 번들 첨부 필수** — `docs/projects/plans/p0-design-v0/` 의 `index.html` / `tokens.css` / `styles.css` / `src/*.jsx` / `chats/chat1.md` 전체. v0의 시각·인터랙션·hypothesis 띠 구조를 그대로 재사용하기 위해 base로 던진다.
3. 본 문서의 **Master Prompt v0.1** 섹션을 복붙
4. 결과 번들을 `docs/projects/plans/p0-design-v0.1/`로 저장 후 NOVA-STATE 갱신

## Master Prompt v0.1

```
첨부한 v0 번들(awm/project/*)을 base로 받는다.
v0는 PRD 11화면 중 핵심 3가설(H1·H2·H3)을 16화면으로 시각화했다.
이번 v0.1 라운드는 v0에서 의도적으로 빠진 보조 화면을 채우는 작업이다.
v0의 토큰(Wanted LaaS Pretendard 기반) · 컴포넌트 · 사이드바 hypothesis 띠 ·
페르소나 토글 · 워크스페이스 셀렉터 · 카드 / chip / table / timeline · 다크 모드 ·
mock 데이터의 anonymization 규칙을 그대로 따른다.

작업 범위는 다음 4개 묶음이다.

═══════════════════════════════════════════
[1] H4 · 5분 first-value 온보딩 (5화면, hypothesis 띠 추가)
═══════════════════════════════════════════

가설: "AI 도구 connect → 첫 세션 import → Today 행 1개 표시가 5분 안에
끝나면, 디자인 파트너는 트라이얼 시작 후 결제 결정에 자율적으로 진입한다."

페르소나: Admin (워크스페이스 생성자 = Admin)
검증 지표: 온보딩 완료 시간 (median) — — → ≤ 5분
사이드바 그룹 라벨: "H4 · 5분 first-value 온보딩"
H4 활성 시 페르소나 자동 스왑: Admin

화면 1 — 워크스페이스 생성
- 입력: 워크스페이스 이름, 세그먼트(B2B SaaS / D2C 이커머스 / 서비스업 / 솔로 인디 / 기타),
        팀 규모(1 / 2~10 / 11~30 / 31~50 / 51+), 한국어 명/요금 통화(KRW 기본)
- 우측 패널: "왜 묻는가" 설명(세그먼트·규모는 디자인 파트너 추적·자동 제안용,
            외부 공유 안 함)
- 하단: "다음 — 도구 연결 →" CTA (primary)
- 진행 게이지 1/5 표기

화면 2 — AI 도구 connect
- 선택형 카드 4개: Claude Code / Cursor / Codex / ChatGPT
  각 카드: 로고 placeholder + 한 줄 설명 + 연결 상태(미연결 / 연결 / 오류)
- "연결" 클릭 → OAuth 미리보기 모달 (도구별 권한 스코프 요약: 세션 메타·코드 변경·명령 로그)
- 좌측 패널: 우리는 *원문 transcript 저장 안 함*, 메타·코드 변경 요약만 저장 — 명시
- 최소 1개 연결 시 "다음 — 첫 세션 import →" 활성
- 진행 게이지 2/5

화면 3 — 첫 세션 import 진행
- 라이브 진행 카드 (Cursor 연결 가정): 단계 4개 — 권한 확인 → 메타 fetch →
  파일 변경 매칭 → 위험 분석. 각 단계 spinner / check 토글
- 우측: 막 import된 세션 미리보기 (의도 / 변경 파일 수 / 위험 유무 chip)
- 하단: "Reviewer 지정 →" CTA — 이 단계는 비동기로 백그라운드 완성, 사용자는 다음으로
- 진행 게이지 3/5

화면 4 — Reviewer 지정
- "AI 변경 검토자(Reviewer)를 최소 1명 지정해 7대 원칙 §1 거버넌스를 활성화하세요" 헤더
- 워크스페이스 멤버 카드 리스트 + 본인은 자동 후보, 멤버 초대 메일 입력 필드
- 역할 라디오: Operator / Reviewer / Admin (역할 의미 1줄씩)
- 우측: 7대 원칙 §1 "거버넌스" 항목이 현재 'pending' — Reviewer 지정 시 'ok'로 바뀌는 미리보기
- 진행 게이지 4/5

화면 5 — 완료 → Today
- 큰 체크 + "온보딩 완료 — 첫 세션이 Today에 도착했습니다"
- 막 import된 세션 카드 1개 (의도·변경 파일·위험·세션 ID) 강조 표시
- 시간 메트릭: "온보딩 시작 → 첫 세션 표시 = 4분 38초" (검증 지표 실측 예시)
- CTA: "Today 열기 →" (H1 today 화면으로 점프)
- 진행 게이지 5/5 + done

[2] Workspace · Members & Roles (3화면, Supporting — hypothesis 띠 없음)
═══════════════════════════════════════════

화면 6 — Members 목록

화면 7 — Member 초대 모달
- 이메일 다중 입력 (chip)
- 역할 사전 지정 라디오 (Operator / Reviewer / Admin)
- 한국어 자동 메시지 미리보기 (편집 가능, mock 발신자: 워크스페이스 Admin)
- "원문 transcript 저장 안 함" 안내 1줄 (signup 직후 인지)
- 좌하단: 초대 메일 카피 예시 — 1인 운영 사실 명시 라인 포함

화면 8 — Roles & Risk 룰
- 매트릭스: 행 = 위험 카테고리 8개(DB / Secret/Env / Deploy/Infra / Destructive /
            Auth / Migration / Large Diff / Failed Verification),
            열 = 역할 3개(Operator / Reviewer / Admin)
- 각 셀: "검토 권한" / "승인 권한" / "view-only" / "—" chip
- 하단 알림: "역할 변경 시 audit log에 기록 — 변경 사유 입력 권장"

[3] Settings 화면군 (4화면, Supporting)

사이드바 그룹 라벨: "Settings"

화면 9 — Profile & Account
- 본인 이름·이메일·역할·가입 일·알림 채널(이메일 / Slack / 채널톡)
- 비밀번호 변경 / SSO 연결 / API Key 발급(워크스페이스 단위)
- 데이터 다운로드(GDPR / K-PIPA 준용 — 본인 워크스페이스 데이터 JSON export)
- 계정 삭제 (위험 액션, 확인 모달 inline preview)

화면 10 — Integrations
- 연결된 AI 도구 (H4와 동일 4개) 상태 카드 + 마지막 동기 시각 + "재연결 / 끊기"
- 외부 서비스: GitHub (repo selector), Slack (워크스페이스 + 채널 매핑), 채널톡
- 신규 통합 placeholder: Jira / Notion (지원 예정 chip — 임의 기간 표시 X)

화면 11 — Notifications
- 알림 규칙 표 — 행: 이벤트 종류(고위험 신호 / 미설명 세션 / 체인 무결성 깨짐 /
                       Reviewer 응답 지연 / 비용 한도 임박),
                       열: 채널(이메일 / Slack / 채널톡 / 인앱), 셀: ON/OFF 토글
- 무음 시간대 설정 (1인 운영 sustainability)
- 미리보기: 고위험 신호 알림 mock 메시지

화면 12 — Audit Export Settings
- PDF 양식 선택: 인공지능기본법 7대 원칙 양식 (기본) / 분기 감사 양식 / 커스텀
- 보존 기간: 5년 (법정 권고 — 기본) / 7년 / 10년 / 영구. 변경 시 비용 영향 한 줄
- 해시 알고리즘: SHA-256 (기본, 변경 불가) — 변조 불가 설계임을 명시
- 자동 export 일정: 분기 / 월 / off
- 좌측: 최근 export 5건 mini list (날짜 / 양식 / 페이지 / 검증 상태 chip)
```
