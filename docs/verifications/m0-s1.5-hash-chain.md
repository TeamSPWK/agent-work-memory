## 발견 이슈 (조치)

### Medium #1 — AuditTrail.tsx fallback 정합성 (별도 backlog)

`web/src/screens/audit/AuditTrail.tsx:13` — `ingest.auditEvents.length > 0 ? ingest.auditEvents : AUDIT_EVENTS` 패턴. `awm serve` 미실행 또는 fetch 실패 시 seed `AUDIT_EVENTS`의 fake hash가 fallback으로 노출. S1.6 발견 #4의 *코드 로직*은 해소(useIngest 어댑터가 chain hash 정상 매핑), *UI fallback 정합성*은 별도 작업.

처리 옵션 (backlog로 이연):
- A) fallback 사용 자체를 제거 + "데이터 미연결" 빈 상태 표시
- B) seed AUDIT_EVENTS hash 필드를 `(unconfigured)` placeholder로 변경
- C) ingest.error 상태를 AuditTrail이 명시 노출 (현재 useIngest는 error 필드 보유)

**왜 별도**: S1.5 scope는 *chain 무결성 구현*이며 *UI fallback 디자인*은 S1.6 baseline 발견들과 함께 *S2 측정 후 데이터 기반 우선순위 결정* 항목(NOVA-STATE Blocker #3).

### Low #1 — IngestWorkPacket 미사용 정리 (조치 완료)

useIngest.ts에서 IngestWorkPacket 타입 + IngestData.workPackets 필드 제거. workPackets는 ingest.json에 여전히 노출되지만 hook 어댑터는 audit chain만 사용 (Sessions/Today 화면은 별도 경로). tsc/build 회귀 없음.
