### Ingest raw schema vs UI 어댑터 사용 비율

ingest.json sessions[N] 객체의 keys 25개:

```
actor, agentSummary, commitCandidates, confirmedCommits, endedAt, evidence,
explainBack, flowSteps, fullIntent, id, intentSummary, linkedCommits,
parentSessionId, repo, repoRoot, segmentCount, segmentIndex, sourceKind,
sourceLabel, startedAt, status, title, tool, unresolved, workBrief
```

useIngest.ts `toSessionSeed` 매핑에서 사용:

```
id, tool, startedAt → when, actor, repo, intentSummary → intent,
commitCandidates → files count, status → state / explained
```

→ **7/25 (28%)** 사용. **18 fields 미활용**. UI에 *깊은 메타*가 안 보임.

특히 *회상 도구로서 가장 중요*한 fields가 버려짐:
- `agentSummary` — AI가 만든 세션 요약
- `workBrief` — 작업 브리프
- `explainBack` — 의도/결과/검증/미해결/핸드오프 5필드 (있다면)
- `flowSteps` — 세션 안의 단계별 흐름
- `evidence` — 증거 그래프 raw
- `unresolved` — 미해결 항목

## 발견 7가지 (사용자 화면에서 직접 인지 가능)

1. **Sessions intent가 대화 단편** — 7/30이 20자 미만, *작업 의도* 아닌 *답글*. H1 1분 회상 직접 위협.
2. **매칭이 시간 근접에 의존** — 4 세션이 동일 3 commit. P1 4축 가중 점수가 *시간*만 dominant. H2-a TP 위협.
3. **어댑터가 데이터 깊이 평탄화** — useIngest가 25 fields 중 7만 사용. agentSummary/workBrief/explainBack/flowSteps/evidence 통째로 unused.
4. **Audit hash chain fake** — UI는 마치 있는 것처럼 표시하나 코드는 더미. PRD §5.5 미구현(S1.5)을 UI에서 가시화.
5. **Risk 신호 세션 fan-out 부재** — 세션 30건 모두 risk:null. workPackets riskCount>0 4건의 정보가 세션·Risk Radar로 전파 X.
6. **repo 파서 정합 90%** — `05/12`, `2026-05-12/new-chat` 같은 날짜·세션 경로 파편 3건.
7. **WorkPacket summary 미요약** — agentSummary 있음에도 intentSummary 원문이 그대로 들어감.
